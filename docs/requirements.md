# 功能需求

## 项目名称

项目中文名：高寒智建：面向高寒地区建筑图纸生成与审核的智能体平台

工程代号：ArcticCAD-Agent

高寒智建不是传统 CAD 软件的简单替代品，而是围绕高寒地区建筑图纸生成、识别、审核与修改构建的 AI 智能体平台。系统前期以 Web 工作区和 JSCAD 代码驱动图纸为核心，后续通过导出与桥接能力兼容 AutoCAD/DWG 工作流。

## 工程组织约束

当前工程采用以下目录结构：

```text
ArcticCAD-Agent/
  frontend/    # Vue3 + Vite + shadcn-vue 工作台
  backend/     # OpenHarness 改造后的 CAD Agent 后端
  docs/        # 需求、架构、技术路线、MVP、UI 和 JSCAD skill 文档
```

文档、接口和后续开发任务均以该结构为准。前端和后端需要通过明确 API 与事件协议协作，避免前端直接耦合 OpenHarness 内部实现。

## 项目背景

高寒地区建筑设计与施工需要处理低温、冻融、保温、风雪、临建施工效率等特殊约束。普通地区标准建筑图纸往往无法直接适用于高寒环境，需要专业设计人员反复修改。施工单位获取图纸后，也常需要专家进行二次审查，以确认图纸是否满足高寒地区建筑规范和施工要求。

现有 CAD 设计工具专业门槛较高，普通建筑从业者难以快速完成图纸生成、修改和审查工作。高寒智建希望利用大模型代码生成、视觉识别、工具调用、思维链任务分解和规范知识接口，形成“画图、改图、识图、审图”一体化智能工作流。

## 项目研究目的和意义

高寒地区建筑设计面临标准图纸适配困难、图纸审查依赖专家经验、CAD 工具使用门槛较高等问题。普通地区建筑图纸往往无法直接应用于低温、冻融、风雪和保温要求更高的高寒环境，需要专业人员反复修改，导致设计周期长、成本高、效率低。同时，施工单位在图纸使用前通常还需依赖专家进行二次审查，审查结果容易受经验差异影响，难以保证全面性和一致性。

本项目拟构建“高寒智建：面向高寒地区建筑图纸生成与审核的智能体平台”，围绕识图、审图、画图、改图四项核心能力，利用大模型代码生成、视觉理解、工具调用和规范知识接口，实现高寒建筑图纸的智能生成、快速修改和辅助审核。系统将降低 CAD 设计使用门槛，缩短高寒建筑图纸设计与审查周期，提高图纸质量和规范符合度，为黑龙江省智能建造产业集群建设提供技术支撑，推动高寒地区建筑业向数字化、智能化方向转型，具有显著的经济价值和社会价值。

## 目标用户

- 高寒地区建筑设计人员。
- 智慧工地技术人员。
- 施工单位图纸审查人员。
- 需要快速生成临建、节点、构件草图的项目团队。
- 非 CAD 专业背景但需要表达建筑构造意图的一线工程人员。

## 核心能力

- 画图：用户用自然语言描述需求，智能体生成 JSCAD 代码，写入项目工作区并触发前端渲染。
- 改图：用户提出修改需求，或系统基于审图结果修改已有 JSCAD 代码，生成新版本并保留历史。
- 识图：调用 GLM 视觉模型识别当前画布截图、上传图片或导入图纸预览图，输出结构化图纸理解结果。
- 审图：结合 JSCAD 代码、渲染截图、几何信息、用户要求和知识库接口生成审查意见。
- 导出：MVP 支持 Web 端导出 SVG、STL、DXF，后续桥接 AutoCAD/DWG。

## 接口分离需求

- 前端可在 mock API 下独立开发工作台、代码编辑器、渲染反馈、事件流和审图 UI。
- 后端可在无前端情况下通过 API 测试完成 agent 流程。
- 前端只通过 `AgentGateway` 等 API client 与后端通信。
- 后端业务层只依赖领域接口，不直接写死 DeepSeek、GLM、知识库、文件系统或 AutoCAD 实现。
- JSCAD 执行错误必须通过统一 `JscadRunResult` 回传。
- 知识库、模型、存储、导出均可替换。
- OpenHarness 作为后端基座，不能泄漏为前端依赖。

## JSCAD 代码闭环要求

智能体生成的 JSCAD 代码不能只停留在聊天文本中，必须进入工作区和版本系统：

1. DeepSeek 根据用户需求生成 JSCAD 代码。
2. 系统将代码写入当前项目文件，如 `main.js`，并生成版本号。
3. 前端在浏览器沙箱中执行 JSCAD 代码并渲染几何结果。
4. 前端截获语法错误、JSCAD API 调用错误、几何返回错误、渲染错误和导出错误。
5. 错误日志、当前代码和原始用户需求通过 `JscadRunResult` 回传给智能体。
6. 智能体基于错误信息最小修改代码，生成修复版本。
7. 修复版本再次写入、渲染和验证。

MVP 至少支持一次自动修复循环，并保留修复前后的代码版本。

## MVP 范围

- Vue3 Web 工作台。
- shadcn-vue 紧凑式 SaaS 界面。
- JSCAD 代码编辑、写入、执行、渲染和报错截获。
- DeepSeek 对话、JSCAD 代码生成和代码修复。
- GLM 视觉识图和审图。
- JSCAD skill 与 API 白名单约束。
- 知识库 mock 接口。
- 项目版本和运行产物归档。
- 基础 SVG/STL/DXF 导出入口。

## 非 MVP 范围

- 完整 DWG 原生编辑器。
- 完整高寒建筑规范库。
- 多人协同和 BIM 全流程。
- 复杂多 Agent 协作系统。
- AutoCAD COM 自动化强依赖。
